以下の PRD / Todo をレビューしてください。

レビュー対象:
- 実装タスクの細かい文言ではなく、全体設計・実装順序・依存関係・検証可能性

観点:
- 要件漏れや成功条件の不足
- 制約や非目標との矛盾
- 実装順序や依存関係の破綻
- テスト、ビルド、リント、移行作業の不足
- 過剰設計、危険な前提、後戻りしやすい設計

Red Team 観点:
- この計画が失敗する最もありそうな理由は何か
- 見積もりやスコープが楽観的すぎないか
- より単純な代替案がないか
- 実装開始前に人間が明示的に承認すべきリスクは何か

出力形式:
# Review

## Summary

## Critical Issues
- issue

## Design Risks
- risk

## Suggested Changes
- change

## Red Team Challenges
- challenge

ルール:
- 同じ論点を言い換えて何度も指摘しない
- 些末な文言修正より、実装上の問題を優先する
- 問題がなければ、そのまま実装に進める理由を明示する
