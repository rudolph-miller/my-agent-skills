以下の PRD と Todo をレビューしてください。

観点:
- 要件の抜け漏れ
- 実装リスク
- 設計の矛盾
- テスト観点の不足
- Todo 分解の粗さまたは過不足
- 依存関係や実装順序の問題

Red Team 観点（批判的レビュー）:
- 前提条件への疑義: 暗黙の前提を洗い出し、「本当にそうか？」を問う
- 楽観的見積もりの検出: スコープ・工数・期待効果が甘くないか
- 代替アプローチ: 他の設計・実装方針がないか、なぜこの案が最善か
- 失敗シナリオ: この PRD 通りに実装した場合に起きうる最悪のケース

出力形式:
# Review

## Summary

## Issues
- issue
- issue

## Suggestions
- suggestion
- suggestion

## Test Considerations
- test
- test

## Red Team Challenges
- 前提への疑義や代替案、失敗シナリオを記述
